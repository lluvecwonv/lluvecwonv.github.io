# OLMo: Accelerating the Science of Language Models

**저자:** Dirk Groeneveld, Iz Beltagy, Pete Walsh, Akshita Bhagia 외 다수
**소속:** Allen Institute for AI (AI2), University of Washington, Yale University 등
**학회:** ACL 2024 (Long Paper)
**코드:** https://allenai.org/olmo

## 한 줄 요약

언어 모델의 과학적 연구를 가능하게 하기 위해, 모델 가중치·학습 데이터·학습 코드·평가 프레임워크·중간 체크포인트를 **모두** Apache 2.0으로 공개한 진정한 의미의 오픈 언어 모델 프레임워크.

---

## 1. 논문 개요

최근 가장 강력한 언어 모델들은 독점적 인터페이스 뒤에 감춰져 있으며, 학습 데이터·아키텍처·개발 과정의 중요한 세부 사항이 공개되지 않는다. 이는 편향과 잠재적 위험을 포함한 과학적 연구에 치명적인 장벽이 된다. OLMo는 이 문제를 해결하기 위해 AI2에서 구축한 경쟁력 있는 오픈 언어 모델로, 기존의 "오픈" 모델들이 주로 모델 가중치와 추론 코드만 공개하는 것과 달리, 전체 파이프라인을 투명하게 공개한다.

---

## 2. OLMo 프레임워크

### 2.1 모델 아키텍처

OLMo는 디코더 전용 트랜스포머 아키텍처를 채택하며, 1B와 7B 두 가지 스케일을 제공한다. 바닐라 트랜스포머 대비 주요 수정 사항은 다음과 같다:

| 설계 선택 | 설명 |
|-----------|------|
| **바이어스 제거** | LLaMA, PaLM과 동일하게 모든 바이어스 항을 제거하여 학습 안정성 향상 |
| **비모수적 레이어 정규화** | 아핀 변환(adaptive gain/bias) 없는 레이어 노름 사용. 안전성과 속도에서 최적 |
| **SwiGLU 활성화 함수** | ReLU 대신 SwiGLU 사용. 히든 크기 ~8/3 d, 128의 배수로 올림 (7B: 11,008) |
| **회전 위치 임베딩 (RoPE)** | 절대 위치 임베딩을 대체하여 길이 일반화 성능 향상 |
| **수정된 BPE 토크나이저** | GPT-NeoX-20B 기반, PII 마스킹 토큰 추가. 어휘 크기 50,280 (50,304로 패딩) |

**모델 구성 상세 (Table 1):**

| 설정 | OLMo-1B | OLMo-7B |
|------|---------|---------|
| 레이어 수 (L) | 16 | 32 |
| 히든 차원 (D) | 2048 | 4086 |
| 어텐션 헤드 (H) | 16 | 32 |
| 학습 토큰 | 2T | 2.46T |
| Peak LR | 4.0E-4 | 3.0E-4 |
| Warmup | 2000 steps | 5000 steps |
| Weight Tying | yes | no |
| 배치 크기 | ~4M | ~4M |

### 2.2 사전학습 데이터: Dolma

모델 파라미터의 공개는 진전되었지만, 사전학습 데이터셋은 여전히 충분히 개방되지 않은 상태다. 대부분의 오픈 모델(비공개 모델은 말할 것도 없고)은 학습 데이터를 공개하지 않으며, 데이터에 대한 문서화도 연구를 재현하거나 완전히 이해하기에 필요한 수준의 상세함이 부족하다. 이로 인해 학습 데이터가 모델의 능력과 한계에 어떤 영향을 미치는지 연구하는 것이 어려웠다.

이러한 문제를 해결하기 위해, 저자들은 사전학습 데이터셋 **Dolma**를 구축하고 공개했다. Dolma는 (1) 대규모 언어 모델 사전학습에서 일반적으로 사용되는, (2) 일반 대중이 접근 가능한 다양한 데이터 소스에서 수조 개의 토큰과 수십억 개의 문서를 수집한 다양하고 다중 소스 코퍼스이다.

**Dolma 구성 (Table 2):**

| 소스 | 유형 | UTF-8 크기 (GB) | 문서 수 (백만) | 토큰 수 (십억) |
|------|------|-----------------|---------------|---------------|
| Common Crawl | 웹 페이지 | 9,812 | 3,734 | 2,180 |
| GitHub | 코드 | 1,043 | 210 | 342 |
| Reddit | 소셜 미디어 | 339 | 377 | 80 |
| Semantic Scholar | 학술 논문 | 268 | 38.8 | 57 |
| Project Gutenberg | 도서 | 20.4 | 0.056 | 5.2 |
| Wikipedia | 백과사전 | 16.2 | 6.2 | 3.7 |
| **합계** | | **11,519** | **4,367** | **2,668** |

*토큰 수는 GPT-NeoX 토크나이저 기준이다.*

Dolma는 다음의 체계적인 파이프라인으로 구축되었다: (1) 언어 필터링, (2) 품질 필터링, (3) 콘텐츠 필터링, (4) 중복 제거, (5) 멀티소스 혼합, (6) 토크나이징. 각 소스의 문서는 큐레이션 과정과 최종 릴리스 모두에서 별도로 유지된다.

Dolma 보고서(Soldaini et al., 2024)는 설계 원칙, 구축 세부 사항, 그리고 콘텐츠/품질 필터, 중복 제거, 다중 소스 혼합 등 중요한 데이터 큐레이션 실천에 대한 추가 분석 및 실험 결과를 제공한다. 저자들은 고성능 데이터 큐레이션 도구를 오픈소스로 공개했으며, 이 툴킷은 Dolma에 대한 추가 실험, 연구 재현, 새로운 사전학습 코퍼스의 빠른 큐레이션에 활용할 수 있다. 또한 데이터셋 분석을 위한 **WIMBD** 도구(Elazar et al., 2024)도 함께 오픈소스로 공개했다.

### 2.3 적응 (Adaptation)

TÜLU 데이터와 학습 설정을 따라 OLMo를 범용 챗 어시스턴트로 학습:
- **OLMo+SFT:** distilled + human-written instruction 데이터 혼합으로 instruction finetuning
- **OLMo+SFT+DPO:** distilled preference 데이터로 Direct Preference Optimization 추가 정렬

### 2.4 평가

- **온라인 평가:** 학습 중 매 1000 스텝(~4B 토큰)마다 다운스트림 태스크 평가
- **오프라인 평가:** Catwalk 프레임워크(다운스트림) + Paloma 벤치마크(퍼플렉시티)

## 3. 학습 상세

### 3.1 분산 학습 프레임워크

PyTorch FSDP + ZeRO 옵티마이저 전략으로 모델 가중치와 옵티마이저 상태를 GPU 간 샤딩. Mixed-precision training (bfloat16).

### 3.2 옵티마이저

AdamW (Beta1=0.9, Beta2=0.95, Epsilon=1.0E-5). LR warmup 5000 steps, 이후 피크의 1/10까지 선형 감소. Gradient clipping (l²-norm ≤ 1.0).

**핵심 트릭:** 메인 학습 완료 후 마지막 1,000 스텝에서 LR을 0까지 선형 감소 → 퍼플렉시티·엔드태스크 성능 동시 향상.

### 3.3 데이터 준비

Dolma에서 2T 토큰 샘플링. 각 문서 끝 EOS 토큰 추가, 2048 토큰 단위 chunking, 셔플링. 배치 순서와 구성은 공개 아티팩트에서 완전 재구성 가능.

### 3.4 하드웨어

| 클러스터 | GPU | 노드 수 | GPU 메모리 | 인터커넥트 |
|---------|-----|---------|-----------|----------|
| **LUMI** | AMD MI250X | 최대 256 | 128GB | 800Gbps |
| **MosaicML** | NVIDIA A100 | 27 | 40GB | 800Gbps |

### 4.2 내재적 언어 모델링 평가 (Paloma)

**평가 설정**

내재적 평가를 위해 Paloma 벤치마크를 사용한다. Paloma는 개별 도메인별 성능 분석부터 도메인 조합에 대한 요약 결과까지 다양한 수준의 분석을 제안한다. 본 논문에서는 두 가지 수준의 세분화로 결과를 보고한다: (1) Paloma 18개 소스 중 11개에 대한 집계 성능 (Magnusson et al., 2023), (2) 이 11개 소스 각각에 대한 세부 결과.

이 11개 소스 하위 집합은 Paloma에서 공개적으로 사용할 수 없는 소스, 극단적이거나 유해한 텍스트를 포함하는 소스, 또는 Paloma의 오염 제거(decontamination) 방식이 지원하지 않는 코드 데이터로 구성된 소스를 제외한 것이다. 포함된 소스는 다음과 같다:

| 소스 | 유형 |
|------|------|
| C4 (Raffel et al., 2020) | 웹 크롤 |
| mC4-en (Chung et al., 2023) | 다국어 웹 크롤 (영어) |
| Wikitext 103 (Merity et al., 2016) | 위키피디아 |
| Penn Treebank (Marcus et al., 1999) | 구문 분석 코퍼스 |
| RedPajama (Together Computer, 2023) | 다중 소스 |
| Falcon-RefinedWeb (Penedo et al., 2023) | 웹 크롤 |
| Dolma (Soldaini et al., 2024) | 다중 소스 |
| M2D2 S2ORC (Reid et al., 2022) | 학술 논문 |
| M2D2 Wikipedia (Reid et al., 2022) | 위키피디아 |
| C4 100 domains (Chronopoulou et al., 2022) | 도메인별 웹 |
| Dolma 100 Subreddits (Soldaini et al., 2024) | 레딧 |

서로 다른 어휘(vocabulary)를 가진 모델 간의 공정한 비교를 위해, Gao et al. (2020)이 정의한 **bits per byte**를 보고한다.

**결과 (Figure 2):**

![Figure 2: Paloma 11개 평가 데이터 소스에 대한 bits per byte](/images/olmo/figure2_paloma.png)

*Figure 2: Paloma의 11개 평가 데이터 소스와 그 조합에 대한 bits per byte. OLMo의 학습 데이터는 Paloma에 대해 명시적으로 오염 제거(decontamination)되었다.*

**Sources Combined** 결과에서, OLMo-7B는 Paloma의 11개 데이터 소스 조합에 대해 6개의 유사 규모 언어 모델과 비교하여 경쟁력 있는 적합도를 보인다. 특히 학습 데이터가 Paloma에 대해 명시적으로 오염 제거된 점을 감안하면 주목할 만하다.

최종 모델(도형)과 중간 체크포인트(점선)를 통해 비교하면, OLMo 결과는 다른 모델들과 유사한 스케일링 추세를 따른다. 중간 체크포인트의 성능은 해당 체크포인트가 학습률 스케줄의 어디에 위치하는지에 영향을 받는다. 따라서 더 적은 스텝으로 학습된 모델은 반드시 더 높은 샘플 효율성을 가지지 않더라도 더 가파른 학습 곡선을 보이는 경향이 있다. 그럼에도 MPT-7B는 이 집계 결과에서 다른 모델들보다 더 빠르게 개선되는 것으로 나타났다. 이는 사전학습 데이터 구성과 Paloma 도메인과의 일치도(예: MPT는 27% 비-Common Crawl 데이터를 사용하는 반면, LLaMA는 18%, RedPajama는 12.2%, OLMo는 11.2%), 그리고 다양한 데이터 전처리 결정(예: MPT의 C4에 대한 의미적 중복 제거) 등 여러 요인에 기인할 수 있다.

개별 소스별 세부 분석에서 더 큰 변동이 관찰된다:

- **Common Crawl 기반 평가 (C4 등):** OLMo-7B가 좋은 성능을 보임. 단, Common Crawl의 다른 후처리 방식은 해당 데이터로 학습된 모델이 가장 잘 적합함 (예: Falcon-7B의 Falcon RefinedWeb)
- **비-웹 스크래핑 소스 (WikiText-103, M2D2 S2ORC, M2D2 Wikipedia):** OLMo-7B의 샘플 효율성이 다른 모델 대비 낮음
- **RedPajama:** 7개 도메인 중 2개만 Common Crawl이고, Paloma가 각 소스 내 도메인에 동일 가중치를 부여하므로 유사한 패턴을 보임

위키피디아, ArXiv 논문 등 큐레이션된 소스의 이질적 데이터는 스크래핑된 웹 텍스트보다 희소하므로, 사전학습 코퍼스가 확대됨에 따라 이러한 언어 분포에 대한 샘플 효율성을 유지하는 것이 도전 과제가 될 것이다.
