This research note summarizes the EMNLP 2023 paper **G-Eval: NLG Evaluation using GPT-4 with Better Human Alignment**.

**Authors**: Yang Liu, Dan Iter, Yichong Xu, Shuohang Wang, Ruochen Xu, Chenguang Zhu
**Affiliations**: Microsoft Cognitive Services Research
**Paper link**: https://arxiv.org/abs/2303.16634
**GitHub**: https://github.com/nlpyang/geval

## One-line Summary

By having an LLM auto-generate evaluation steps via Chain-of-Thought (CoT), score outputs in a form-filling paradigm, and compute the final score as a **probability-weighted sum of output tokens**, G-Eval substantially improves correlation with human judgments compared to all prior NLG evaluation metrics.

## 1. Introduction

Automatically measuring the quality of text generated by Natural Language Generation (NLG) systems is a challenging problem. With LLMs capable of generating high-quality and diverse texts often indistinguishable from human-written texts, this problem has become even more important.

The limitations of existing automatic evaluation metrics are clear:

- **BLEU, ROUGE, METEOR** and other conventional reference-based metrics have relatively low correlation with human judgments, especially for tasks requiring creativity and diversity.
- These metrics require associated **reference output**, which is costly to collect for new tasks.

Recent studies propose directly using LLMs as reference-free NLG evaluators, scoring candidate outputs based on generation probability. However, meta-evaluations show these LLM-based evaluators still have lower human correspondence than medium-size neural evaluators.

This paper proposes **G-Eval** with three key ideas:

1. **Chain-of-Thoughts (CoT)**: Feed only Task Introduction and Evaluation Criteria to the LLM and have it auto-generate detailed Evaluation Steps.
2. **Form-filling paradigm**: Concatenate the prompt, CoT, and input text, then have the LLM output a score as if filling in a form.
3. **Probability-weighted scoring**: Use output token probabilities to refine discrete scores into fine-grained continuous scores.

### Main Contributions

1. LLM-based metrics generally outperform reference-based and reference-free baseline metrics in terms of correlation with human quality judgments, especially for open-ended and creative NLG tasks such as dialogue response generation.
2. LLM-based metrics are sensitive to instructions and prompts, and chain-of-thought can improve performance by providing more context and guidance.
3. LLM-based metrics can provide more fine-grained continuous scores by re-weighting discrete scores with their respective token probabilities.
4. LLM-based metrics have a potential issue of preferring LLM-generated texts over human-written texts, which may lead to self-reinforcement if used as a reward signal.

## 2. Method

G-Eval is a prompt-based evaluator with three main components.

### 2.1 Prompt for NLG Evaluation

The prompt is a natural language instruction that defines the evaluation task and the desired evaluation criteria. For example, for evaluating coherence in text summarization:

> *"You will be given one summary written for a news article. Your task is to rate the summary on one metric."*

With customized evaluation criteria appended. For coherence:

> *"Coherence (1-5) - the collective quality of all sentences. We align this dimension with the DUC quality question of structure and coherence whereby 'the summary should be well-structured and well-organized. The summary should not just be a heap of related information, but should build from sentence to sentence to a coherent body of information about a topic.'"*

### 2.2 Auto Chain-of-Thoughts for NLG Evaluation

The chain-of-thoughts (CoT) is a sequence of intermediate representations generated by the LLM during the text generation process. Some evaluation criteria need more detailed evaluation instructions beyond a simple definition, and manually designing such steps for each task is time-consuming.

Key finding: **The LLM can generate such evaluation steps by itself.** By adding a line *"Evaluation Steps:"* to the prompt, the LLM automatically generates detailed CoT.

Example — LLM auto-generated CoT for Coherence evaluation:

> *1. Read the news article carefully and identify the main topic and key points.*
> *2. Read the summary and compare it to the news article. Check if the summary covers the main topic and key points of the news article, and if it presents them in a clear and logical order.*
> *3. Assign a score for coherence on a scale of 1 to 5, where 1 is the lowest and 5 is the highest based on the Evaluation Criteria.*

### 2.3 Scoring Function

**Problems with direct scoring**:

1. **Score distribution skew**: For some tasks, one digit dominates the distribution (e.g., 3 on a 1-5 scale), leading to low variance and low correlation with human judgments.
2. **Integer output tendency**: LLMs usually only output integer scores, even when explicitly asked for decimal values, leading to many ties that fail to capture subtle differences between generated texts.

**Solution: Probability-weighted summation**

Given a set of scores $S = \{s_1, s_2, ..., s_n\}$ predefined in the prompt (e.g., 1 to 5), the probability of each score $p(s_i)$ is calculated by the LLM, and the final score is:

$$score = \sum_{i=1}^{n} p(s_i) \times s_i$$

This method obtains more fine-grained, continuous scores that better reflect the quality and diversity of the generated texts.

### Overall Framework (Figure 1)

1. **Task Introduction + Evaluation Criteria** → Input to LLM
2. LLM generates **Auto CoT** (detailed Evaluation Steps)
3. Prompt + CoT + Input Context + Input Target → LLM outputs score in **form-filling** paradigm
4. **Probability-weighted summation** of output tokens as the final score

## 3. Experiments

### 3.1 Implementation Details

- **Models**: OpenAI's GPT family — GPT-3.5 (text-davinci-003) and GPT-4
- **GPT-3.5**: decoding temperature = 0 (deterministic output)
- **GPT-4**: Does not support output of token probabilities, so set $n = 20$, $temperature = 1$, $top\_p = 1$ to sample 20 times to estimate token probabilities
- **Notation**: G-Eval-4 = GPT-4 backbone, G-Eval-3.5 = GPT-3.5 backbone

### 3.2 Benchmarks

Three meta-evaluation benchmarks are used:

**SummEval** (Fabbri et al., 2021): A summarization evaluation benchmark built on CNN/DailyMail. Human ratings on four aspects: `fluency`, `coherence`, `consistency`, and `relevance`.

**Topical-Chat** (Mehri and Eskenazi, 2020): A testbed for meta-evaluating dialogue response generation evaluators. Four aspects: `naturalness`, `coherence`, `engagingness`, and `groundedness`. Turn-level evaluation.

**QAGS** (Wang et al., 2020): A benchmark for evaluating hallucinations in summarization. Measures the `consistency` dimension on two datasets: CNN/DailyMail and XSum.

### 3.3 Baselines

- **N-gram based**: ROUGE-1, ROUGE-2, ROUGE-L
- **Embedding based**: BERTScore, MoverScore
- **Learning based**: BARTScore, UniEval, FactCC, QAGS (metric), CTC
- **LLM based**: GPTScore
- **Dialogue specific**: USR

## 4. Analysis

### 4.1 Does G-Eval Prefer LLM-based Outputs?

A key concern when using an LLM as an evaluator: **the LLM may prefer outputs generated by the LLM itself over human-written texts.**

The authors investigated this using a dataset from Zhang et al. (2023), where freelance writers produced human summaries and GPT-3.5 generated summaries for comparison, with human annotators judging preference.

The data was divided into three categories: (1) human summaries rated higher by human judges, (2) GPT-3.5 summaries rated higher, and (3) rated equally good.

**Results** (Figure 2):
- G-Eval-4 assigned higher scores to human summaries when human judges also preferred them, and lower scores when humans preferred GPT-3.5 summaries — consistent with human judgment so far.
- **However**, G-Eval-4 **always** gave higher scores to GPT-3.5 summaries than to human summaries, even when human judges preferred the human-written ones.

**Potential reasons**:
1. NLG outputs from high-quality systems are inherently difficult to evaluate. The original paper found inter-annotator agreement (Krippendorff's alpha) to be very low at 0.07.
2. G-Eval may have a bias toward LLM-generated summaries because the model could share the same concept of evaluation criteria during both generation and evaluation.

**Implication**: If LLM-based evaluation is used as a reward signal for further tuning, this could result in **self-reinforcement** — over-fitting to the LLM's own evaluation criteria rather than the true evaluation criteria of the NLG tasks.

### 4.2 The Effect of Chain-of-Thoughts

Comparing G-Eval-4 with and without CoT on SummEval (Table 1), G-Eval-4 with CoT has higher correlation on all dimensions, especially for `fluency`. CoT provides more context and guidance for the LLM to evaluate generated text and helps explain the evaluation process and results.

### 4.3 The Effect of Probability Normalization

Comparing G-Eval-4 with and without probability normalization on SummEval (Table 1):

- **Kendall-Tau ($\tau$)**: The version without probabilities appears higher, but this is because many ties are neither counted as concordant nor discordant, which does not reflect the model's true evaluation capacity.
- **Spearman ($\rho$)**: The probability-weighted version is higher. Since Spearman is rank-based, fine-grained continuous scores better capture subtle differences between generated texts.

### 4.4 The Effect of Model Size

G-Eval-4 showed higher correlation than G-Eval-3.5 on most dimensions and datasets (Tables 1 and 3), except for `engagingness` and `groundedness` on Topical-Chat. This demonstrates that **larger model size improves G-Eval performance**, especially on more challenging and complex evaluation tasks such as `consistency` and `relevance`.

## 5. Related Work

### N-gram based Metrics
BLEU calculates the geometric mean of modified n-gram precision with a brevity penalty. ROUGE is a recall-oriented metric for summarization evaluation. More than 60% of recent NLG papers rely only on ROUGE or BLEU for evaluation, but these metrics fail to measure content quality or capture syntactic errors.

### Embedding-based Metrics
BERTScore measures the similarity between generated and reference texts based on contextualized BERT embeddings. MoverScore improves BERTScore by adding soft alignments and new aggregation methods.

### Task-specific Evaluators
FactCC is a BERT-based classifier predicting factual consistency of summaries. QAGS is a QA-based evaluator that generates questions from summaries and checks if answers can be found in the source document. USR assesses dialogue response generation from different perspectives.

### Unified Evaluators
UniEval uses a pretrained T5 model to encode evaluation tasks as QA tasks, capable of evaluating different aspects of text generation.

### LLM-based Evaluators
GPTScore evaluates texts with generative pre-training models like GPT-3, assuming they assign higher probability to high-quality generated text. Unlike G-Eval, GPTScore formulates evaluation as a conditional generation problem instead of a form-filling problem.

## 6. Conclusion

This paper proposes G-Eval, a framework for using LLMs with Chain-of-Thoughts (CoT) to evaluate the quality of generated texts.

**Key results**:
- On two NLG tasks — text summarization and dialogue generation — G-Eval outperformed all prior SOTA evaluators and achieved higher human correspondence.
- G-Eval with GPT-4 achieved an average Spearman correlation of **0.514** on SummEval.
- The authors conducted analysis on LLM-based evaluator behavior and highlighted the potential issue of bias toward LLM-generated texts.

**Limitations and implications**:
- The bias of LLM-based evaluators preferring LLM-generated text may lead to self-reinforcement problems.
- GPT-4's lack of token probability access requires sampling 20 times to estimate probabilities, incurring additional cost.
- As model size significantly impacts evaluation capability, stronger future LLMs are expected to further improve evaluation performance.
