This research note summarizes the SIGIR 2025 paper **Template-Based Financial Report Generation in Agentic and Decomposed Information Retrieval**.

The authors are Yong-En Tian, Yu-Chien Tang, Kuang-Da Wang, An-Zi Yen, and Wen-Chih Peng from National Yang Ming Chiao Tung University.

The core research question is:

**"When generating structured template-based financial reports with LLMs, which approach is more effective: multi-agent methods (AgenticIR) or prompt decomposition methods (DecomposedIR)?"**

Noting the limitations of existing earnings call summarization methods—which tend to produce broad and shallow summaries—this is the first study to formalize the task of **template-based report generation**, which is widely used in industry practice, and systematically compare two major approaches.

Paper link: https://doi.org/10.1145/3726302.3730253
Project: https://github.com/bryant-nn/Template-Based-Financial-Report-Generation

## TL;DR

While the multi-agent framework (AgenticIR) offers flexibility, the decomposed approach using prompt chaining to decompose templates into sub-queries (DecomposedIR) demonstrates **27% higher average performance** across all four key evaluation characteristics—providing an important industry insight that agent autonomy is not always optimal.

## 1. Introduction — Why Template-Based Financial Report Generation Matters

For financial analysts, writing comprehensive financial reports based on earnings call transcripts is a core task. These reports are applied across a wide range of fields including comparative financial analysis across companies, stock movement prediction, risk assessment, and sentiment analysis. However, identifying key financial evidence from long earnings call transcripts and organizing them into structured reports is a time-consuming and challenging task.

Existing approaches have framed this as an **earnings call transcript summarization** task, but these methods generate broad and comprehensive summaries without providing the **detailed, section-specific information** that financial analysts actually need. The authors reframe this problem as a **template-based question-answering** task, aiming to extract information tailored to each topic in a concise manner.

The paper makes three contributions:

1. Formalize the template-based financial report generation task and systematically compare two mainstream industry approaches (AgenticIR, DecomposedIR)
2. Conduct experiments on both financial domain datasets and the weather domain (SumIPCC) dataset to confirm cross-domain generalization potential
3. Provide insights for practitioners regarding agent framework deployment

## 2. Related Work

Research on leveraging LLMs as autonomous agents is active. CAMEL introduces a role-playing framework that enables autonomous collaboration between agents, while AutoGen provides a multi-agent conversation framework. LLMCompiler strengthens agent capabilities through parallel function execution optimization.

Goldsack et al. explores the impact of multi-agent frameworks on earnings call transcript summarization, closely related to this paper. However, while Goldsack et al. focused on the text content generated by agents themselves, this paper compares **the effects and differences of RAG using AgenticIR and decomposed prompting** from the perspective of template-based report generation.

Research on improving summarization strategies is also active. Gamage et al. propose a framework where multiple LLMs autonomously retrieve knowledge to generate decision-support responses, while Suresh et al. present a modularized approach that decomposes the summarization task into prompt-based sub-queries and integrates them with RAG.

## 3. Method

### 3.1 Data Collection and Preprocessing

Through pilot exploration, the authors selected **5 major semiconductor manufacturers** (TSMC, Intel, Samsung, GlobalFoundries, UMC) and collected earnings call documents from 2021-2024 via the Discounting Cash Flows API, building a dataset of **74 financial documents**.

These documents consist of two types:

- **Earnings call transcripts**: Voice recordings of earnings presentations. Divided into 1,000-character chunks with 200-character overlap between consecutive chunks to maintain contextual consistency
- **Financial statements**: Structured in JSON format including balance sheets, cash flow statements, and income statements. These supplement the numerical data retrieved from transcripts

Additionally, the authors collected **financial report templates** from financial analysts and applied them throughout the experiments.

### 3.2 AgenticIR

AgenticIR is a multi-agent framework composed of **6 agents and 2 core functions**. Implemented based on AutoGen, the agents collaborate to generate structured financial reports.

![Figure 2: AgenticIR framework architecture — collaboration structure of User Proxy Agent, Assistant Agent, Financial Retrieval Agent, Financial Manager Agent, User Agent, and Task Decompose Agent](/images/papers/template-financial-report/figure2_agenticIR_framework.png)
*Figure 2: The overall structure of the AgenticIR framework. The Retrieve_Generate() function on the left enables the Financial Retrieval Agent and Financial Manager Agent to search for relevant chunks from earnings call transcripts and process financial information. The Task_Decompose() function on the right allows the User Agent and Task Decompose Agent to decompose complex tasks into sub-tasks.*

**Roles of the 6 agents:**

- **User Proxy Agent**: Receives the user's report generation prompt and pre-defined template and passes them to the Assistant Agent
- **Assistant Agent**: Analyzes the input and selects appropriate function calls to return to the User Proxy
- **Financial Retrieval Agent**: Takes a query q and retrieval count n, and searches for top-n transcript chunks based on semantic similarity using an embedding model
- **Financial Manager Agent**: Synthesizes retrieved chunks and financial statements to generate the final output
- **User Agent**: Submits complex tasks to the Task Decomposition Agent
- **Task Decomposition Agent**: Decomposes tasks into manageable sub-tasks

**2 core functions:**

- **task_decompose**: Decomposes complex tasks into sub-tasks
- **retrieve_generate**: Searches for relevant chunks based on sub-task queries and generates answers

**Workflow**: When the User Proxy Agent receives the report prompt and template, the Assistant Agent analyzes it and performs appropriate function calls. This iterative exchange continues until all sub-tasks are completed, and finally the Assistant Agent compiles the summarized report.

### 3.3 DecomposedIR

DecomposedIR follows a **prompt chaining workflow** that decomposes the report template into section-specific sub-queries (SQ).

![Figure 3: Example of decomposed questions derived from DecomposedIR — each section and subsection is transformed into specific sub-queries (SQ)](/images/papers/template-financial-report/figure3_decomposed_questions.png)
*Figure 3: Example of decomposed questions. The P&L highlights of Section 1 are subdivided into SQ1–SQ3 for Sub Section 1.1 (revenue results) and SQ4–SQ6 for Sub Section 1.2 (wafer sales). Each SQ includes placeholders like "#TIME#" to query information for specific quarters.*

**Operating mechanism:**

1. The LLM decomposes the report template into section-specific sub-queries (SQ)
2. For each SQ, the same embedding model as AgenticIR is used to search for top-n relevant chunks
3. The embedding model encodes each chunk and sub-query into 768-dimensional vectors and determines relevance using cosine similarity
4. The retrieved n chunks and financial statements are passed to the LLM generator
5. The LLM generates answers for each SQ by combining textual context from transcript chunks with precise numerical data from financial statements
6. Once all SQs are processed, results are aggregated and summarized by section according to the report template structure

Key difference: While AgenticIR gives the agent the entire template and lets it decompose autonomously, DecomposedIR provides **explicit structural guidance** for step-by-step decomposition.

## 4. Experiments

### 4.1 Experimental Setup

- **LLM**: Both AgenticIR and DecomposedIR use **GPT-4o-mini**
- **Embedding models**: `fin-mpnet-base` for financial datasets, `stella_en_1.5B_v5` for SumIPCC dataset
- **Retrieval count**: Standardized at n=3 for consistency

### 4.2 Dataset: Finance

**Evaluation Metrics**: Due to the high cost of collecting expert-authored template-based financial reports as references, the authors adopt two reference-free evaluation metrics:

- **DecompEval (DE)**: Prompts an LLM to evaluate whether each sentence aligns with specific characteristics individually, then aggregates results
- **G-Eval (GE)**: Prompts an LLM to assign scores from 1 to 5, generating scores for each characteristic

Following the methodology of Goldsack et al., evaluation uses **4 key characteristics**:

1. **Financial Takeaways**: Key financial figures and statistics for the relevant quarter
2. **Financial Context**: Insights on current financial performance compared to the previous quarter
3. **Reasoning Correctness**: Accuracy of reasoning and explanations regarding financial performance for the relevant quarter
4. **Management Expectation**: Management's outlook and expectations for the next quarter

**Self-reflection mechanism**: Both frameworks include a self-reflection (w/ sr) variant. This mechanism enables agents to analyze generated content, check alignment with each characteristic, and adjust answer strategies.

### 4.3 Quantitative Results

![Table 1: Financial dataset results — DecomposedIR surpasses AgenticIR on all metrics](/images/papers/template-financial-report/table1_financial_results.png)
*Table 1: Results on the financial dataset. DE denotes DecompEval and GE denotes G-Eval. Highest scores are in bold, second-best scores are underlined.*

**Key Findings:**

- **DecomposedIR consistently outperforms AgenticIR on all metrics.** Quantitatively, DecomposedIR demonstrates an average 27% higher performance across all four characteristics (p < 0.05 by Pearson's χ² test).
- **Self-reflection improves both DecompEval and G-Eval by an average of 9.8%.** AgenticIR also shows slight performance improvements.
- This validates the value of decomposing report templates into detailed sub-queries through prompt chaining.

### 4.4 Readability Analysis

![Table 2: Readability comparison between both methodologies — DecomposedIR generates more complex and longer reports](/images/papers/template-financial-report/table2_readability.png)
*Table 2: Readability comparison between both methodologies on the financial dataset. #Sents denotes average number of sentences, FKGL is Flesch-Kincaid Grade Level, CLI is Coleman-Liau Index, and ARI is Automated Readability Index.*

Evaluation using three readability metrics (FKGL, CLI, ARI), which are calculated based on linguistic characteristics such as character count, syllable count, word count, and sentence count. Higher scores indicate more complex text.

Key observations:

- **Higher DE/GE scores correlate with increased report complexity.** In other words, frameworks with better performance generate more complex reports.
- **Self-reflection increases not only complexity but also report length.** More detailed and contextually rich content may be more suitable for specialized readers.
- However, there are questions about whether statistical measures like syllable counts are optimal for measuring readability of financial reports, a concern also noted in prior research.

### 4.5 Dataset: SumIPCC

To verify cross-domain generalization capability, experiments were also conducted on the **SumIPCC dataset**. SumIPCC comprises 140 summaries derived from lengthy IPCC (Intergovernmental Panel on Climate Change) reports, reorganized into 7 reports.

![Table 3: SumIPCC dataset results — DecomposedIR achieves best performance on both ROUGE and BERTScore](/images/papers/template-financial-report/table3_sumipcc.png)
*Table 3: Results on the SumIPCC dataset. Since ground truth is available, ROUGE and BERTScore (based on deberta-v3-large) are used as evaluation metrics.*

**Key Findings:**

- **DecomposedIR achieves best performance on both ROUGE and BERTScore**, with 33% advantage over AgenticIR on ROUGE-1 and 6.3% advantage on BERTScore
- This suggests that DecomposedIR enables more accurate content retrieval and summarization even in different domains
- **AgenticIR's autonomous query decomposition strategy lacks explicit structural guidance**, which may lead to inconsistencies and performance degradation

**Effect of Self-reflection**: On SumIPCC, DecomposedIR with self-reflection shows 45% advantage over AgenticIR on ROUGE-1 and 8.3% advantage on BERTScore. However, it was also observed that since each report in this dataset comprises sections/subsections with different focuses and styles, self-reflection can create misalignment between generalized guidance and specific section requirements.

## 5. Conclusion

The main findings of this research comparing two LLM-based workflows (AgenticIR, DecomposedIR) are twofold:

**First, decomposed prompt chaining outperforms the multi-agent approach.** DecomposedIR was superior in both evaluation scenarios, demonstrating that AgenticIR has limitations in bridging the gap between unstructured context and structured report templates.

**Second, self-reflection improves report quality but reduces readability.** While report quality on key characteristics improves, increased report complexity has negative effects on readability.

Suggested future research directions include strengthening multi-agent systems using Query-Focused Summarization and applying Aspect-based Summarization to template-based report generation.

## Personal Commentary

This paper illustrates well the fundamental trade-off between **agent autonomy versus explicit structuring**.

AgenticIR, based on AutoGen with 6 agents collaborating autonomously, offers a flexible structure. Yet precisely because of this flexibility, it fails to precisely follow the structured template. In contrast, DecomposedIR achieves consistently superior results through explicit query decomposition using relatively simple prompt chaining methodology.

This has important practical implications. Currently, there is a tendency in industry to regard "agents" as a panacea, but **more simple and structured approaches can actually be more effective depending on task characteristics**. Particularly for tasks like financial report generation that require structured output, following predefined structure proves more reliable than autonomous agent decomposition.

However, there are some limitations worth noting. First, since only GPT-4o-mini was used, generalization to other LLMs requires additional validation. Second, evaluation relies on automatic metrics (DecompEval, G-Eval), and the correlation with actual financial analyst judgment is uncertain. Third, there is insufficient analysis of whether the fixed retrieval count of n=3 is optimal.

Nevertheless, this is valuable research in that it formalizes the **practical task of template-based report generation** and fairly compares two mainstream approaches. Notably, the cross-domain experiments on the SumIPCC dataset demonstrate that these findings are not limited to the financial domain.
