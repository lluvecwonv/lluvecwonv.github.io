# Counterfactual Memorization in Neural Language Models

**논문:** Zhang et al. (2023) | NeurIPS 2023
**저자:** Chiyuan Zhang, Daphne Ippolito, Katherine Lee, Matthew Jagielski, Florian Tramèr, Nicholas Carlini
**소속:** Google Research, CMU, Google DeepMind, ETH Zürich
**arXiv:** [2112.12938](https://arxiv.org/abs/2112.12938)

## 한 줄 요약

언어모델이 사전학습 데이터를 기억하는가? 이것을 *어느 정도나* 측정할 것인가는 어렵다. 대부분의 기존 메트릭은 "일반적인" 기억화(중복된 텍스트, 템플릿, 공개 지식)를 포착한다. 이 논문은 **counterfactual memorization**을 제안한다—심리학의 에피소딕 기억 vs. 의미 기억 구분에서 영감받은 원칙적인 프레임워크다.

핵심 통찰: 무작위 부분집합으로 여러 모델을 학습한 후 특정 예제가 포함된 경우 vs. 제외된 경우 모델의 예측이 얼마나 변하는지 측정한다. 400개의 T5-base 모델을 학습하여 저자들은 **중복과 counterfactual memorization 간의 역의 상관관계를 발견했다**(r = -0.39)—생성 시점 메트릭이 보여주는 것과는 반대다. 또한 **counterfactual influence**를 도입하여 어떤 학습 예제가 특정 예측에 가장 많이 영향을 미치는지 추적한다.

## 1. 서론: 기억화 문제

![Figure 1: 학습 예제의 per-token accuracy — IN 모델(해당 예제 포함) vs. OUT 모델(미포함), RealNews·C4·Wiki40B:en 3개 데이터셋. 대각선 위의 점은 예제가 학습에 포함되었을 때 성능이 향상됨을 나타낸다.](/images/papers/counterfactual-memorization/fig1_scatter_all.png)

언어모델이 사전학습 데이터의 일부를 기억한다는 것은 명백하다. 이는 *정확한* 기억화—모델이 사전학습에서 긴 문자열을 그대로 재현할 수 있는 능력—에서 가장 분명하게 드러나며, 이는 개인정보보호 및 저작권 문제를 야기한다. 하지만 "기억화"는 정확히 무엇을 의미하는가?

기존 메트릭(추출 공격, 다음 토큰 예측, 중복 기반 휴리스틱 등)은 보통 모델이 자주 나타나거나 집중된 형태로 나타나는 정보를 학습했는지 측정한다. 어떤 문단이 학습 데이터에 100번 나타나면, 물론 모델은 그것을 "안다". 문제는 드물고 특정한 정보는 어떤가? 단 한 번만 나타나는 문서는 어떤가?

저자들은 기억화의 인과적 개념이 필요하다고 주장한다. **에피소딕 기억**(특정하고 문맥에 의존적인 사건)과 **의미 기억**(일반 지식)을 구분하는 심리학 연구에서 영감받아, 다음을 제안한다:

- **Counterfactual memorization (CM)**: 특정 학습 예제를 학습 집합에서 제거하면 모델의 행동이 *얼마나* 변하는가?
- **Counterfactual influence (CI)**: 어떤 학습 예제가 주어진 테스트 시점 예측에 영향을 미치는가?

이것은 기억화를 분포 질문("이 텍스트가 일반적인가?")에서 인과 질문("이 특정 예제가 중요한가?")으로 재구성한다.

## 2. 관련 연구

논문은 여러 연구 흐름 내에 자신을 위치시킨다:

1. **신경망의 기억화**: 선행 연구는 모델이 일반적 패턴과 드문 예제 모두를 기억할 수 있음을 보였지만, 둘을 구분하는 메트릭은 제한적이다.

2. **개인정보보호 공격 및 추출**: Carlini et al. 등이 모델이 학습 데이터를 생성하도록 유도될 수 있음을 입증했다. 하지만 추출 메트릭은 "기억화"를 직접 측정하지 않는다—회수 가능성을 측정한다.

3. **영향 함수(Influence Functions)**: Koh & Liang (2017)은 모델 예측을 학습 예제로 추적하기 위해 영향 함수를 제안했다. 이 연구는 이를 인과적, 학습-부분집합-독립적 프레임워크로 확장한다.

4. **에피소딕 vs. 의미 기억**: 심리학 프레이밍은 기계학습에서 새로운 것이다. "이 정보가 흔하지 않은가"를 측정하는 것이 "이 특정 예제가 모델을 형성했는가"를 측정하는 것과 같지 않은 이유를 동기부여한다.

여기서의 신규성은 **분포적 대리(distributional proxies)에 의존하기보다는 counterfactual memorization을 기저-진실 인과 개념으로 제안하는 것**이다.

## 3. 방법론: Counterfactual Memorization 정의 및 추정

### 3.1 형식적 정의

학습 집합 S, 모델 아키텍처 A, 메트릭 M(예: 토큰 단위 정확도)에 대해:

```
mem(x) = E[M(A(S), x) | x ∈ S] - E[M(A(S), x) | x ∉ S]
```

여기서:
- **첫 번째 항**: x가 학습 집합에 포함될 때 예상되는 메트릭
- **두 번째 항**: x가 학습 집합에 포함되지 *않을* 때 예상되는 메트릭
- **차이**: x 포함의 인과적 효과

x를 포함하는 것이 x에 대한 모델 성능을 상당히 향상시킨다면, 그것은 "counterfactually memorized"된 것이다.

### 3.2 부분집합 학습을 통한 추정

이를 직접 계산하려면 모든 가능한 학습 집합에 대해 모델을 재학습해야 하는데, 이는 불가능하다. 대신:

1. **m개의 독립적인 모델을 학습**한다 전체 학습 집합의 무작위 부분집합에서 (각 부분집합은 ~25% 예제를 샘플링)
2. **모델을 두 그룹으로 분할**한다:
   - **IN models**: 부분집합에 예제 x를 포함하는 모델
   - **OUT models**: 부분집합에서 예제 x를 제외하는 모델
3. **mem(x)를 추정**한다 이 두 그룹 간의 평균 성능 차이로

**핵심 효율성**: m개 모델을 한 번만 학습하고, 모든 학습 예제에 대해 counterfactual memorization을 계산한다. 예제별 재학습이 필요 없다.

### 3.3 구현 세부사항

**데이터셋 및 전처리:**

| 데이터셋 | 출처 | 특성 |
|---------|------|------|
| **RealNews** (Zellers et al., 2019) | 뉴스 기사 | 저널리즘 텍스트, 도메인 다양성 |
| **C4** (Raffel et al., 2020a) | 웹 크롤링 | 일반 웹 텍스트, 가장 다양한 분포 |
| **Wiki40B:en** (Guo et al., 2020) | 위키피디아 | 백과사전식 텍스트, 상대적으로 깨끗한 분포 |

계산 비용을 절감하고 데이터셋 간 직접 비교를 가능하게 하기 위해, 각 데이터셋의 학습 집합을 **처음 2^21개 문서로 축소(truncate)** 하여 사용한다.

**모델 학습 설정:**

| 항목 | 세부 사항 |
|------|----------|
| **아키텍처** | Transformer 기반, T5-base (decoder-only) 동등 구조 |
| **파라미터 수** | ~112M |
| **모델 수 (m)** | 400개 (각 데이터셋별) |
| **부분집합 크기** | 학습 집합의 ~25% (무작위 샘플링) |
| **옵티마이저** | Adam (Kingma and Ba, 2015) |
| **학습률** | 0.1 |
| **Weight decay** | 10^-5 |
| **학습 에포크** | 60 |
| **메트릭** | 토큰 단위 정확도 (per-token accuracy) |

**수렴 성능:**

| 데이터셋 | 학습 세트 평균 정확도 | 검증 세트 평균 정확도 |
|---------|---------------------|---------------------|
| **C4** | 44.21% | 27.90% |
| **RealNews** | 47.59% | 31.09% |
| **Wiki40B:en** | 66.35% | 49.55% |

평균적으로 **에포크 5 부근에서 과적합(overfitting)이 시작**되며, 이는 검증 정확도가 감소하기 시작하는 시점으로 확인된다.

이 설계는 우아하다: 400개 모델이 가능한 모델 인스턴스의 분포를 제공하고, 각 예제 x는 자연스럽게 ~100개 IN 모델과 ~300개 OUT 모델에 나타나, 안정적인 추정을 제공한다.

### 3.4 해시 기반 부분집합 샘플링 (Hash-Based Subsampling)

400개 모델 각각이 독립적인 무작위 부분집합에서 학습되어야 한다. 그러나 대규모 텍스트 데이터의 데이터 로딩 API (Tensorflow Datasets, TFDS)는 일반적으로 순차적 접근만 지원하며, 인덱스 목록 기반의 부분집합 로딩을 직접 지원하지 않는다. 단순한 구현 — 현재 예제의 인덱스가 부분집합 인덱스 목록에 있는지 확인 — 은 매우 느리고 부분집합 크기에 따라 확장성이 좋지 않다.

이 문제를 해결하기 위해 논문은 **해시 기반 부분집합 샘플링 predicate**를 구현한다:

**핵심 아이디어:**

전체 학습 예제 수 N, 기대 부분집합 크기 n < N일 때:

1. 각 예제의 인덱스 i를 **N/n개의 해시 버킷**에 매핑한다
2. **특정 하나의 버킷**에 해당하는 모든 예제를 선택한다
3. 각 모델이 독립적인 부분집합을 얻도록, **모델 인덱스(seed)에 기반한 다른 해시 함수**를 사용한다

구체적으로, `uint64` 타입에 대한 알려진 해시 함수와 모델 인덱스 기반의 간단한 의사 난수를 조합하여 구현한다:

```python
def hash_sampler(mod, seed, system):
    """해시 기반 부분집합 샘플러.
    Args:
        mod: total_n_egs // subset_size
        seed: 다른 seed → 다른 부분집합 샘플
        system: 'np' 또는 'tf'
    """
    np_hash = hash_uint64_builder('np')
    mul, offset, remainder = np_hash(seed + 1234 + np.arange(3))
    remainder = remainder % mod
    # ...tf_filter 또는 np_sampler 반환
```

해시 함수 자체는 [잘 알려진 uint64 해시](https://stackoverflow.com/questions/664014/)를 사용한다:

```python
def hash_uint64(x):
    x = uint64_cast(x)
    x = op_xor(x, op_rshift(x, 30)) * uint64_cast(0xbf58476d1ce4e5b9)
    x = op_xor(x, op_rshift(x, 27)) * uint64_cast(0x94d049bb133111eb)
    x = op_xor(x, op_rshift(x, 31))
    return x
```

**검증 결과 (논문 Figure 11):**

1. **샘플 크기 일관성:** 해시 기반 샘플러는 항상 n에 가까운 수의 포인트를 샘플링하며, 분산이 작다
2. **선택 확률 분포:** r = 0.25로 설정했으므로, 각 포인트가 전체 모델에 포함되는 경험적 비율은 평균 0.25여야 한다. 해시 기반 샘플러의 확률 분포가 `numpy.random.choice`와 매우 일치함을 확인
3. **쌍별 독립성 (Pairwise independence):** 두 서로 다른 학습 포인트 x1, x2가 모두 IN이거나 모두 OUT일 확률이 0.625 (= r² + (1−r)²)에 가까운지 측정. 해시 기반 샘플러의 독립성이 `numpy.random.choice`와 유사함을 확인

이 해시 기반 접근법의 **장점**: 각 예제를 개별적으로 효율적으로 평가할 수 있어, 대규모 데이터셋에서도 빠르게 부분집합을 결정할 수 있다. 부분집합 크기가 정확히 n이 되는 것이 보장되지는 않지만, 이 설정에서는 문제가 되지 않는다.

## 4. Counterfactual Memorization 분석

### 4.1 예제 전체의 기억화 분포

![Figure 2: Counterfactual memorization(X축)과 simplicity(Y축, 전체 모델 평균 정확도)의 결합 분포, RealNews·C4·Wiki40B:en 3개 데이터셋. 중간 난이도 예제가 가장 높은 기억화를 보인다.](/images/papers/counterfactual-memorization/fig2_mem_simplicity_all.png)

놀라운 발견: **기억화는 균등하게 분포하지 않는다**. 대신, "중간 난이도" 피크가 있다:

- **쉬운 예제**: 낮은 기억화 (모델이 그 특정 예제 없이도 배움)
- **중간 난이도 예제**: 높은 기억화 (예제가 학습을 위해 중요)
- **어려운 예제**: 낮은 기억화 (IN 모델도 배울 수 없음)

이는 인간의 기억을 반영한다: 당신은 자명한 사실을 "기억"하지 않는다 (모두가 안다), 하지만 특정하고 어느 정도 놀라운 정보를 기억한다. 도움을 받아도 배우기 어려운 극도로 모호한 데이터는 낮은 기억화를 보인다.

### 4.2 도메인별 분석

![Figure 3: 도메인별 기억화 분석. (a) RealNews 도메인별 95-퍼센타일 기억화 vs. 예제 수, (b) C4 동일 분석, (c) RealNews 대표 도메인 기억화 분포, (d) C4 대표 도메인 기억화 분포.](/images/papers/counterfactual-memorization/fig3_url_analysis_all.png)

예제를 도메인별로 분석하면 흥미로운 패턴을 드러낸다:

| 도메인 | 특성 | 기억화 수준 |
|--------|------|---------|
| **reuters.com** | 대형 출판사, 저널리즘 규범 | 낮음 |
| **digitallibrary.un.org** | 다국어, 기술적, 구조화됨 | 높음 |
| **zap2it.com** | TV 목록, 경직된 포맷 | 높음 (구조화됨) |
| **hotair.com** | 평론, 다른 출처 인용 | 낮음 |

핵심 통찰: **구조화된 데이터, 다국어 텍스트, 고유한 포맷**은 중복이 나타나더라도 counterfactually memorized되는 경향이 있다. 대조적으로, **대형 다양 출판사**는 편집 템플릿으로 낮은 기억화를 보인다 (문맥이 예측을 돕는다).

### 4.3 수렴, 학습 역학, 중복

![Figure 4: (a) m개 모델 간의 기억화 순위 Spearman 상관계수 — ~192개에서 수렴. (b) RealNews 학습 에포크별 기억화 분포 — 학습이 진행될수록 증가. (c) 기억화 임계값 이상 예제 비율의 에포크별 변화. (d) 기억화 점수 vs. 근사-중복 수 — 놀라운 음의 상관관계 (r = -0.39).](/images/papers/counterfactual-memorization/fig4_composite.png)

**수렴 (a):** 안정적인 추정을 위해 몇 개의 모델(m)이 필요한가? Spearman 상관계수: m=32에서 ~0.85, m=96에서 ~0.98, m=400에서 ~0.99. 실제로는 ~96-192개 모델이면 충분하지만, 400개는 우수한 정밀도를 제공한다.

**학습 역학 (b, c):** 기억화는 정적이지 않다. Counterfactual memorization은 에포크 1부터 60까지 일반적으로 증가한다. 59%의 예제는 에포크와 함께 일관되게 증가하는 기억화를 보인다. 이는 기억화가 이산적 "학습" 사건보다는 연속적, 누적적 과정임을 시사한다.

## 5. 중복과 기억화: 놀라운 역의 상관관계

### 5.1 핵심 발견

**위 Figure 4의 패널 (d)**가 논문의 가장 놀라운 결과 중 하나를 보여준다:

논문의 가장 놀라운 결과 중 하나:

```
중복 수와 counterfactual memorization 간의
Pearson 상관계수: r = -0.39
```

이는 **음의** 상관관계다. 학습 집합에서 여러 번 나타나는 예제는 *낮은* counterfactual memorization을 보인다.

### 5.2 왜 이것이 생성 시점 메트릭의 반대인가?

선행 연구(예: 추출 공격)는 **중복이 더 추출 가능함**을 보였다. 왜 역전인가?

직관:
- **생성 시점 기억화** ("모델이 이 텍스트를 얼마나 쉽게 생성할 수 있는가?"): 중복이 도움이 된다 왜냐하면 뭔가를 여러 번 보는 것이 모델 예측에서 더 "확고히" 만들기 때문이다.
- **Counterfactual memorization** ("이 특정 복사본이 중요한가?"): 중복이 *덜* 중요하다 왜냐하면 한 복사본을 제거해도 상처가 없다—다른 복사본이 남아있기 때문이다. 5번 나타나는 문서는 한 번 나타나는 문서보다 모델 성능에 대해 복사본당 덜 기여한다.

간단히 말해: **중복은 counterfactual 관점에서는 "중복"이지만, 생성 관점에서는 "끈기 있다"**.

### 5.3 함의

이는 "기억화"가 단일하지 않음을 드러낸다. 다른 메트릭은 다른 현상을 포착한다:
- 모델이 이 텍스트를 출력하는가? (생성 메트릭)
- 이 예제가 예측에 인과적으로 영향을 미치는가? (Counterfactual 메트릭)

저자들은 강조한다: *counterfactual memorization은 기본적으로 다른 유형의 기억화를 측정한다*.

## 6. Counterfactual Influence: 기억화에서 영향으로

Counterfactual memorization은 다른 예제에 의해 전달되지 않는 희귀 정보를 포함하는 학습 예제를 식별한다. 자연스러운 후속 질문은: 모델이 기억화된 예제의 정보를 추론 시 누출하는가? 이전 연구들은 공격자가 특정 예제가 학습 집합에 존재하는지 알아내려 하는 **멤버십 추론 공격**(Shokri et al., 2017; Sablayrolles et al., 2019; Long et al., 2020)을 연구했다. 이 논문은 적대적 공격자가 없는 표준 모델 평가 환경에서, **"특정 학습 예제를 보는 것이 검증 예제에 대한 예측에 강하게 영향을 미치는가?"**를 정량화한다. 다시 말해, 학습 집합의 단일 예제가 검증 예제의 예측에 크고 과대한 영향을 미치는지를 측정한다.

### 6.1 정의

**정의 6.1 (Counterfactual Influence).** 학습 집합 D를 학습된 모델에 매핑하는 학습 알고리즘 A와 성능 측정 M이 주어졌을 때, 학습 예제 x ∈ D가 다른 예제 x'에 미치는 counterfactual influence는 다음과 같다:

```
infl(x ⇒ x') ≜ E_{S⊂D, x∈S}[M(A(S), x')] - E_{S⊂D, x∉S}[M(A(S), x')]    (3)
```

여기서 S는 D에서 샘플링된 학습 예제의 부분집합이다. 기댓값은 S의 무작위 샘플링과 학습 알고리즘 A의 무작위성 모두에 대해 취한다. x'은 검증 세트, 테스트 세트, 생성된 예제, 또는 학습 예제일 수 있다.

경험적 추정은 counterfactual memorization과 유사하게, D에서 m개의 부분집합 S₁, ..., Sₘ을 균일하게 샘플링(|Sᵢ| = r|D|)하여 계산한다:

```
inflc(x ⇒ x') ≜ mean_{i: x∈Sᵢ}[M(A(Sᵢ), x')] - mean_{i: x∉Sᵢ}[M(A(Sᵢ), x')]    (4)
```

이는 학습 샘플 x의 존재가 다른 예제 x'의 예측에 얼마나 영향을 미치는지를 측정한다. 주목할 점: **`mem(x) = infl(x ⇒ x)`** — 즉, counterfactual memorization은 자기-영향(self-influence)이다.

### 6.2 검증 세트 예제에 대한 영향

기억화 추정에 사용된 동일한 모델들로, 식 (4)에 따라 검증 세트에 대한 counterfactual influence를 추정할 수 있다. 검증 세트의 각 예제에 대해, 모든 학습 예제로부터의 영향을 추정한다.

![Figure 4: (a) 특정 테스트 예제에 대한 모든 학습 예제의 영향 분포 히스토그램 (RealNews). 파란색·주황색 예제는 일부 학습 예제로부터 높은·중간 영향을 받으며, 녹색은 무작위 예제로 모든 학습 예제로부터 영향이 거의 0이다. (b) 각 학습 예제의 기억화 점수(X축)와 검증 세트에 대한 최대 영향(Y축)의 결합 분포 — RealNews와 Wiki40b:en. 히스토그램은 분포의 꼬리를 더 잘 시각화하기 위해 로그 스케일이다.](/images/papers/counterfactual-memorization/fig5_mem_infl_main.png)

![Figure 4 (확장): 기억화와 최대 영향의 결합 분포 — RealNews, C4, Wiki40B:en 3개 데이터셋.](/images/papers/counterfactual-memorization/fig5_mem_infl_all.png)

Figure 4(a)는 세 개의 서로 다른 검증 예제에 대해 모든 학습 예제로부터의 영향 분포를 보여준다:

- **녹색 예제** (무작위 선택): 대부분의 검증 예제의 일반적인 행동을 대표한다 — 모든 (개별) 학습 예제로부터 거의 0에 가까운 영향을 받는다.
- **파란색·주황색 예제** (높은·중간 최대 영향을 갖도록 샘플링): 학습 집합에서 하나(또는 소수)의 강한 영향자를 가지며, 히스토그램 오른쪽의 이상치 막대로 나타난다. 나머지 모든 학습 예제로부터는 역시 미미한 영향만 받지만, 녹색 예제보다 분산이 더 크다.

직관적으로, 대부분의 학습 예제는 검증 세트 예제에 작은 영향만 미친다 — 모델이 많은 학습 예제에 걸쳐 공유되는 분포적 패턴을 학습하기 때문이다. 그러나 **높은 counterfactual memorization을 가진 학습 예제 x는 다른 예제와 공유되지 않는 희귀 정보를 포함**한다. 따라서 검증 세트 예제 x'이 유사한 정보를 포함하면, infl(x ⇒ x')이 클 수 있다.

**기억화와 영향의 관계 (Figure 4b):**

Figure 4(b)는 각 학습 예제 x의 mem(x)를 검증 세트 전체에서의 최대 영향 max_{x'} infl(x ⇒ x')에 대해 플롯한다. 직관과 일치하게:

- **작은 기억화 점수를 가진 예제**는 작은 최대-영향 점수를 갖는다.
- 검증 세트에 대한 **더 큰 영향 점수는 일반적으로 더 큰 기억화 점수를 요구**한다 — 학습 예제 자체의 기억화보다 크지 않다.
- 그러나 **큰 기억화를 가진 모든 예제가 큰 영향을 미치는 것은 아니다**. 특히 기억화가 0.4를 초과하는 예제에서 최대-영향이 크게 감소한다. 가능한 이유:
  1. 매우 높은 기억화를 가진 예제의 상당수는 단순히 저품질 텍스트여서, 기억화는 필요하지만 검증 예제에 영향을 줄 만한 흥미로운 정보를 인코딩하지 않는다.
  2. 기억화된 예제가 희귀하고 유용한 정보를 인코딩하더라도, 검증 세트에 관련 문서가 없으면 최대-영향이 낮을 수 있다 — 모든 데이터셋이 학습 세트보다 상당히 작은 검증 세트를 가진다는 점에서 특히 그렇다.

낮은 영향 범위에서는 두 가지 유형의 상관관계가 흔히 관찰된다:
1. **템플릿화된 텍스트** — 유사한 학습 예제가 많아서 영향이 분산됨
2. **표면적으로 관련된 문서** — "ST. CLOUD – This week in our 'Behind the Scenes' series on WJON" 같은 공유 접두사나 "FSIS, the Centers for Disease Control and Prevention" 같은 공통 지식의 부분 문자열로 인한 피상적 관련

**핵심 결론:** 영향은 테스트 시점의 모델 예측을 분석하고 귀속하기 위한 효과적인 도구이다. (Counterfactual) memorization에 의해 획득된 정보에 의존하는 예측에 대해, 어떤 학습 예제가 그 정보를 제공했는지 정확히 식별할 수 있다. 근-중복 학습-검증 문서 쌍의 관찰은 대규모 인터넷 크롤링 텍스트 코퍼스에서 데이터 오염을 식별한 최근 연구(Lee et al., 2021; Dodge et al., 2021)와 일치한다.

### 6.3 생성된 텍스트에 대한 영향

영향 추정은 검증 세트에 국한되지 않는다. 생성된 예제에 대해서도 영향을 추정할 수 있다. 이 섹션에서는 RealNews에서 학습된 **Grover 모델**(Zellers et al., 2019)의 공개 생성물을 평가한다. 구체적으로, **Grover-Mega (p=0.96)** — RealNews 데이터셋에서 학습된 15억 파라미터 모델 — 의 생성물을 사용한다.

![Figure 8: Grover-Mega가 생성한 텍스트에 대한 RealNews 학습 예제의 최대 영향 히스토그램. 학습-검증 영향(Figure 4b)과 비교하면, 최대-영향이 커질수록 히스토그램이 더 빠르게 감소하며, 최대-영향의 값 범위도 두 배 더 작다.](/images/papers/counterfactual-memorization/fig8_grover_influence.png)

Figure 4(b)의 학습-검증 영향과 비교하면, 생성 텍스트에 대한 영향 히스토그램은 최대-영향이 커질수록 **더 빠르게 감소**한다. 또한 최대-영향의 값 범위도 **두 배 더 작다**.

높은 영향을 받는 생성 예제를 많이 발견하지 못한 이유는 두 가지다:

1. **생성물 수의 제한**: 공개 릴리스에는 24,576개의 생성물만 있어, 검증 예제보다 훨씬 적다. 따라서 많은 기억화된 학습 예제의 대응하는 예제가 생성물에서 샘플링되지 않는다. 비교를 위해, 이전 연구(Carlini et al., 2020; Lee et al., 2021)는 생성에서 기억화를 식별하기 위해 100,000개 이상의 예제를 생성했다. 또한 이전 접근법은 학습 세트의 중복을 계산에 포함하지만, counterfactual memorization은 중복을 필터링한다.

2. **학습 집합 범위의 차이**: Grover 모델은 전체 RealNews 학습 세트에서 학습되었지만, 이 분석은 처음 200만 개 학습 예제로 제한했다. 계산에서 누락된 더 많은 높은 영향 학습 예제가 잠재적으로 존재할 수 있다.

3. **경험적 통찰**: 중간 난이도 피크, 도메인 변형, 중복과의 역의 상관관계
4. **Counterfactual influence 프레임워크** 정보 흐름을 추적하기 위해
5. **공개 분석** 재현성을 위해

### 7.2 한계 및 향후 연구

저자들은 중요한 한계를 인정한다:

1. **규모**: T5-base (112M 파라미터)는 현대 LLM (GPT-3, GPT-4)보다 훨씬 작다. 스케일링 행동은 불명확하다.

2. **언어**: 영어만의 연구다. 다국어 패턴은 다를 수 있다.

3. **데이터셋 크기**: RealNews, C4, Wiki40B는 상당하지만 현대 전체 학습 집합보다 작다.

4. **일반화**: 생성 작업에서의 결과 (Grover)는 예비적이다; 생성적 기억화의 완전한 특성화는 열려 있다.

저자들은 다음을 제안한다:
- 더 큰 모델로 스케일링 (분산 학습 필요)
- 버전 관리된 Wikipedia 체계적 패러프레이즈 연구를 위해
- 도메인 시프트 하의 기억화 탐색
- 코드 및 다른 양식에서의 테스트

## 8. 개인 코멘트

이 논문은 나는 정말로 중요하다고 생각하는 개념적 기여를 한다. "일반적" 기억화(중복, 공개 지식)와 "에피소딕" 기억화(특정하고 인과적으로 중요한 예제) 간의 구분은 인간이 기억을 실제로 경험하는 방식과 일치한다. "나는 그것을 기억했다"라고 말할 때, 우리는 보통 특정 사실이 우리를 놀라게 했거나 중요했다는 의미다—100번 들었다는 의미가 아니다.

### 무엇이 공감한다

1. **인과적 프레이밍**: "이 텍스트가 일반적인가?"(분포 질문)가 아니라 "이 예제가 중요한가?"(인과 질문)를 묻는 것이 개념적으로 더 깔끔하다. 개인정보보호 및 귀속 우려를 직접 다룬다.

2. **역 중복 효과**: 이는 정말로 놀라우며 다른 메트릭이 다른 현상을 측정하는 방식을 강조한다. "추출 가능"을 "기억화"와 혼동하는 것에 대해 더 신중하도록 만들어야 한다.

3. **효율성**: 25% 부분집합으로 400개 모델을 학습하는 것은 영리하다. 비싸지만 실행 가능하다—예제별 정확한 재학습보다 훨씬 싸다.

### 보고 싶은 것

1. **스케일링 분석**: Counterfactual memorization이 70B+ 파라미터 모델로 어떻게 스케일되는가? 같은 패턴이 유지되는가?

2. **명령어-튜닝 효과**: 대부분의 현대 LLM은 명령어 데이터로 미세 조정된다. CM이 미세 조정 중에 변하는가? RLHF는 어떤가?

3. **더 긴 형태 기억화**: 논문은 토큰 단위 정확도를 측정한다. 다중-토큰 범위나 전체-문서 일관성은 어떤가?

4. **개인정보보호 함의**: Counterfactual memorization이 개인정보보호 누출과 상관관계가 있다면, 이는 더 표적화된 데이터 제거 전략을 안내할 수 있다.

5. **패러프레이즈 저항**: 모델이 여러 패러프레이즈 전체에서 개념을 기억할 수 있는가? 이는 CM이 정말로 "에피소딕" 기억에 대한 것인지 아니면 추상 지식에 대한 것인지를 말한다.

### 광범위한 중요성

이 연구는 중요한 교차점에 앉아 있다: **LLM의 기계론적 이해**, **개인정보보호 우려**, 그리고 **귀속/저작권**. 언어모델이 더 강력해지면서, 그들이 학습 데이터에서 무엇을 흡수했는지—그리고 특히, 특정 예제가 제거되면 무엇을 잃을지—를 이해하는 것은 책임감 있는 배포를 위해 중요해진다.

Counterfactual 프레임워크는 후속 연구에 영감을 줄 가능성이 높다. 다음으로 확장될 것을 예상한다:
- 더 큰 모델에서 영향 추적
- CM을 모델 편집 기법과 연결
- 영향 추정을 사용하여 학습 데이터 선택
- 전략적 데이터 증강을 통해 기억화에 대항
